# T-003 — Journal module

**Status**: backlog
**Type**: feat

---

## Goal

Build the journal module: daily entries with rich text editing via TipTap, a list view, and full CRUD via Server Actions.

---

## Acceptance criteria

- [ ] `/journal` route lists all journal entries (date descending)
- [ ] `/journal/new` opens a new entry with today's date pre-filled
- [ ] `/journal/[id]` opens an existing entry for editing
- [ ] TipTap editor with starter-kit, placeholder, and character-count extensions
- [ ] Entries saved via Server Action (no direct DB calls from client)
- [ ] Entry title is optional; date is required
- [ ] Entry content stored as TipTap JSON in `journal.content`
- [ ] `entity` row created on journal save (type: "journal")
- [ ] Content embedded via `text-embedding-3-small` on save (async, non-blocking)
- [ ] Delete entry with confirmation dialog
- [ ] Zod schema in `src/modules/Journal/schemas/` is single source of truth for form types
- [ ] All components follow file structure rules
- [ ] All tests green
- [ ] `npm run build` clean
- [ ] `npm run lint` clean
