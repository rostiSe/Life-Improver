# T-001 — Design token system

**Status**: backlog
**Type**: feat

---

## Goal

Establish the complete design token system as the foundation for all future UI work. Every colour, spacing, typography, and radius value must be a named token in `globals.css`. No component ever references a raw value.

---

## Acceptance criteria

- [ ] All semantic colour tokens defined in `globals.css` under `:root` (light) and `[data-theme="dark"]` (dark)
- [ ] All spacing tokens defined as CSS custom properties
- [ ] All typography tokens (size, weight, line-height) defined
- [ ] All border-radius tokens defined
- [ ] Tailwind `@theme inline` block maps every token to a Tailwind utility class
- [ ] `next-themes` wired into `layout.tsx` using `attribute="data-theme"` — NOT class-based
- [ ] Theme toggle component written in `src/components/ThemeToggle/`
- [ ] ThemeToggle has `index.tsx`, `index.styles.ts`, `index.test.tsx`, `index.stories.tsx`
- [ ] One story per variant (at minimum: icon-only, icon + label)
- [ ] All tests green
- [ ] `npm run build` clean
- [ ] `npm run lint` clean
- [ ] Light and dark mode visually verified (screenshot or manual check)
