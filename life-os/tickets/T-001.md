# T-001 — Design token system

**Status**: done
**Type**: feat

---

## Goal

Establish the complete design token system as the foundation for all future UI work. Every colour, spacing, typography, and radius value must be a named token in `globals.css`. No component ever references a raw value.

---

## Acceptance criteria

- [x] All semantic colour tokens defined in `globals.css` under `:root` (light) and `[data-theme="dark"]` (dark)
- [x] All spacing tokens defined as CSS custom properties
- [x] All typography tokens (size, weight, line-height) defined
- [x] All border-radius tokens defined
- [x] Tailwind `@theme inline` block maps every token to a Tailwind utility class
- [x] `next-themes` wired into `layout.tsx` using `attribute="data-theme"` — NOT class-based
- [x] Theme toggle component written in `src/components/ThemeToggle/`
- [x] ThemeToggle has `index.tsx`, `index.styles.ts`, `index.test.tsx`, `index.stories.tsx`
- [x] One story per variant (at minimum: icon-only, icon + label)
- [x] All tests green
- [x] `npm run build` clean
- [x] `npm run lint` clean
- [x] Light and dark mode visually verified (screenshot or manual check)
