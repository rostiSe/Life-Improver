# T-013 — Auth: login / logout

**Status**: backlog
**Type**: feat

---

## Goal

Secure the application with Supabase Auth (email/password). Single user MVP — no registration flow needed, the account is pre-created in the Supabase dashboard.

---

## Acceptance criteria

- [ ] `/login` route with email + password form
- [ ] Form uses React Hook Form + Zod for validation
- [ ] Login calls Supabase Auth `signInWithPassword` via a Server Action
- [ ] On success, redirects to `/` (dashboard)
- [ ] On failure, shows an error message (invalid credentials)
- [ ] All routes except `/login` are protected via Next.js middleware
- [ ] Middleware reads Supabase session cookie and redirects unauthenticated users to `/login`
- [ ] Logout button in the navigation shell calls `signOut` Server Action and redirects to `/login`
- [ ] Supabase client utility in `src/lib/supabase/` — server and browser clients separated
- [ ] No auth state stored in Zustand — session lives in Supabase cookie only
- [ ] All tests green
- [ ] `npm run build` clean
- [ ] `npm run lint` clean
